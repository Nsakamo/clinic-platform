# Codex → Claude 引き継ぎ（右腕くん）

## 今回の目的

右腕くん単体でSlackエスカレーションを完結させ、うけつけるん連携テナントだけスタッフAIチャットから
患者・予約情報を参照し、安全な二段階確認付きでキャンセル・日時変更を行えるようにする。

## 実装内容

- 設定画面にSlack Incoming Webhook、通知ON/OFF、テスト通知を追加。
- Webhook値はレスポンスへ返さず、`CRED_KEY` 設定時は暗号化保存。
- AIの要対応/緊急判定、予約自動受付のスタッフ引き継ぎ、送信失敗を右腕くんから直接Slack通知。
- 同一受信メッセージの重複通知を防ぎ、`PUBLIC_BASE_URL` 設定時は該当会話への直接リンクを付与。
- スタッフAIチャットに、本人確認済みの現在患者だけを対象とした予約一覧・空き枠照会を追加。
- キャンセル/日時変更は `propose` までで停止。対象患者・予約・内容を確認カードへ表示し、明示承認後だけ `confirm`。
- 患者確認待ち・別のスタッフ確認待ちがある場合は新しい書き込み操作を停止。
- 確認カードを閉じても再表示でき、期限切れ依頼は破棄。
- 予約の一回限りの実行命令は患者向け恒久メモへ保存しない。
- 設定画面のSlack欄に、Slack App作成・Incoming Webhook有効化・通知先チャンネル選択・テスト・保存までの
  折りたたみ式ガイドと公式管理画面リンクを追加。Webhookをチャットやメールへ貼らない注意も画面内に表示。
- Slack Appのボット未作成エラーを避けるため、Incoming Webhookを追加する前に `OAuth & Permissions` の
  Bot Token Scopesへ `incoming-webhook` を追加する手順と、既存インストール時の再インストール手順を追記。

## 環境変数

- `PUBLIC_BASE_URL`: 右腕くんの公開URL。Slackの会話直接リンクに使用。無ければ通知は届くがリンクは省略。
- `CRED_KEY`: 既存資格情報とSlack Webhookの保存時暗号化。32バイト（hex64桁またはbase64）。
- `PLATFORM_SECRET`, `PARTNER_BOOKING_URL`: うけつけるん連携。右腕くん単体テナントのSlackには不要。

## Claude E2E確認項目

1. 右腕くんstagingの設定画面でWebhookを直接入力し、テスト通知がテスト専用Slackへ届く。
2. 再度設定を開いてもURL値は表示されず「設定済み」だけが見える。
3. `PUBLIC_BASE_URL` を設定し、要対応の「テスト」問い合わせ通知から同じ会話が開く。
4. 同じ受信イベントで複数の通知が届かない。次の新着では新しい通知が届く。
5. うけつけるん未連携テナントでLINE/メール・AI・Slackが動き、予約操作は実行されない。
6. 連携テナントのテスト患者で「予約を確認」「YYYY-MM-DDの空き枠」を試し、最新情報が表示される。
7. 「この予約をキャンセル」「YYYY-MM-DD HH:MMへ変更」で確認カードが出るが、承認前は予約が変わらない。
8. カードの患者名・予約日時・内容を確認し、「実行しない」は無変更、「実行」は1回だけ反映される。
9. 別患者の予約ID、操作不可予約、患者確認待ちとの競合、期限切れ、二重クリックが安全に拒否される。
10. 右腕くん由来の通知が、うけつけるん側Slackから重複して届かない。
